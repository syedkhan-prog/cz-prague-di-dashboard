# Prague DI Dashboard - Tourist / Local / Expat

Single-page DI overview for **Bolt Food, Prague (city_id 271)**.
Splits delivered orders into 3 cohorts based on phone prefix + home_city, and
shows Bolt-funded discount intensity (DI%) for previous week, last week, and MTD.

> **Live dashboard:** https://syedkhan-prog.github.io/cz-prague-di-dashboard/

## Cohorts

| Segment   | Definition |
|-----------|------------|
| Local     | `phone_anonymized = '420'`                                              |
| Expat     | `phone_anonymized != '420'` AND `home_city_id = 271`                    |
| Tourist   | `phone_anonymized != '420'` AND (`home_city_id != 271` OR NULL)         |
| Unknown   | NULL phone                                                              |

DI% = `SUM(campaign_spend_bolt_eur) / SUM(order_gmv_eur)` on delivered orders
in Prague (excludes provider co-fund).

## How refresh works

The "Refresh Now" button behaves differently depending on where you're viewing it:

- **On the deployed GitHub Pages site** -> opens the
  [GitHub Actions "Run workflow" page](https://github.com/syedkhan-prog/cz-prague-di-dashboard/actions/workflows/refresh.yml)
  in a new tab. Click **Run workflow -> Run workflow** there, wait ~3 minutes,
  then come back and reload this page.
- **On the local server** (`python cz_dashboard.py --serve`) -> hits the local
  `/refresh` endpoint, which re-queries Databricks immediately and reloads.

The GitHub Action also runs **automatically every day at 06:00 UTC**
(08:00 Prague summer / 07:00 winter), so the deployed dashboard self-heals
even if no one hits the button.

## Local development

```bash
pip install -r requirements.txt

# Use cached data, just open the page locally
python cz_dashboard.py --serve --no-query

# Force a fresh query (OAuth browser sign-in to Databricks)
python cz_dashboard.py --serve --refresh

# Just rebuild static HTML+JSON without serving (this is what CI does)
python cz_dashboard.py --build --refresh --no-browser

# Backfill / pretend "today" is some other date
python cz_dashboard.py --serve --refresh --date 2026-04-26
```

Auth is auto-detected:
- `env.DATABRICKS_TOKEN` set -> PAT (used by GitHub Actions)
- otherwise -> OAuth browser flow (used locally)

## First-time GitHub setup

1. Create the repo (public, so GitHub Pages + Actions are free):
   ```bash
   gh repo create cz-prague-di-dashboard --public --source=. --remote=origin --push
   ```
2. Add the Databricks token as a secret:
   ```bash
   gh secret set DATABRICKS_TOKEN --body 'dapiXXXXXXXXXXXXXXXXXXXXXXXXXXXX'
   ```
3. Enable GitHub Pages from `/docs` on `main`:
   ```bash
   gh api -X POST repos/{owner}/{repo}/pages -f source[branch]=main -f source[path]=/docs
   ```
4. Trigger the first refresh:
   ```bash
   gh workflow run refresh.yml
   ```
   (or from the Actions tab in GitHub).

The "Refresh Now" button on the deployed dashboard points at this same workflow.

## Source tables

- `ng_delivery_spark.dim_order_delivery` - order facts (`country_code='cz'`, `city_id=271`, `order_state='delivered'`)
- `ng_public_spark.user_user` - user lookup for `phone_anonymized` + `home_city_id`
